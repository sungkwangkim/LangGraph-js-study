# LangChain vs LangGraph: 내 챗봇에 맞는 도구 선택하기

챗봇 개발 시 LangChain과 LangGraph 중 어떤 것을 선택해야 할지 고민되시나요? 결론부터 말씀드리면, **챗봇의 복잡도에 따라 선택이 달라집니다.**

- **빠른 프로토타입과 표준 RAG 챗봇**이 필요하다면 **LangChain**이 좋습니다.
- **스스로 판단하고, 검색 결과를 평가하며, 대화 흐름을 제어하는 똑똑한 챗봇**이 필요하다면 **LangGraph**가 정답입니다.

---

## 🤖 챗봇에게 맛집을 추천받는 두 가지 방법

"잠실 맛집 알려줘" 라는 동일한 질문에 대해 LangChain과 LangGraph가 어떻게 다르게 반응하는지 비유를 통해 알아봅시다.

### 방법 1: ⛓️ LangChain - 정해진 레시피대로 움직이는 요리사

LangChain은 정해진 순서대로만 작동하는 **'잘 짜인 레시피'** 와 같습니다. `[대화기록 확인] -> [문서 검색] -> [답변 생성]` 이라는 고정된 파이프라인을 따릅니다.

> **User:** "잠실 맛집 알려줘."
>
> **LangChain Bot:**
>
> 1.  **(고정)** 이전 대화 기록을 확인합니다.
> 2.  **(고정)** "잠실 맛집" 관련 문서를 검색합니다.
> 3.  **(고정)** 검색된 정보를 바탕으로 답변을 생성합니다.
>
> **결과:** "잠실 맛집으로는 'A 식당'이 있습니다." (검색 결과가 좋든 나쁘든 그대로 전달)

**핵심:** 코드가 간결하고, 표준적인 RAG(검색-증강-생성) 흐름을 매우 빠르게 만들 수 있습니다.

### 방법 2: 📈 LangGraph - 스스로 생각하고 판단하는 미식가

LangGraph는 단순히 레시피를 따르는 것을 넘어, 각 단계의 결과물을 **스스로 평가하고 최상의 결과를 위해 행동을 수정** 하는 **'미식가'** 와 같습니다.

> **User:** "잠실 맛집 알려줘."
>
> **LangGraph Bot:**
>
> 1.  "잠실 맛집" 관련 문서를 검색합니다.
> 2.  **(판단)** "A 식당 리뷰가 별로네. 이건 좋은 추천이 아니야."
> 3.  **(행동 수정)** 질문을 "리뷰 좋은 다른 잠실 맛집"으로 재구성하여 다시 검색합니다.
> 4.  **(재평가)** "B 식당은 리뷰가 훌륭하군! 이걸 추천해야겠다."
> 5.  최종 답변을 생성합니다.
>
> **결과:** "리뷰가 좋은 잠실 맛집으로 'B 식당'을 추천합니다."

**핵심:** 상태(State), 노드(Node), 조건부 엣지(Conditional Edge)를 통해 **'순환(Loop)'** 과 **'분기(Branching)'** 가 가능하여, 스스로 판단하고 수정하는 복잡하고 능동적인 로직을 구현할 수 있습니다.

---

### 🤔 한 눈에 보는 비교

| 기준          | ⛓️ LangChain                      | 📈 LangGraph                               |
| :------------ | :-------------------------------- | :----------------------------------------- |
| **목표**      | 정보 검색 및 답변 (표준 RAG)      | 스스로 판단하고 수정하는 에이전트          |
| **흐름**      | **선형적 (A -> B -> C)**          | **순환적, 조건부 (A -> B? -> C? -> A)**    |
| **제어**      | 단순하고 빠름. 흐름이 고정됨.     | 복잡하지만 유연함. 상태 기반 제어.         |
| **핵심 비유** | 정해진 레시피를 따르는 **요리사** | 결과를 평가하고 행동을 수정하는 **미식가** |

### 🚀 최종 추천 (=== 저의 추천)

1.  **먼저 LangChain으로 핵심 기능을 빠르게 완성.**
2.  이후 챗봇을 더 고도화하고 싶을 때, 예를 들어 "검색된 맛집의 리뷰가 너무 적으면 다른 곳을 다시 찾아줘" 또는 "사용자가 '별로야'라고 답하면 자동으로 다음 맛집을 추천해 줘" 같은 **셀프 수정(Self-Correction) 기능** 을 추가하고 싶을 때 **LangGraph 도입을 고려** 하는 것이 좋을것 같아요.
